# 欧加真智能调光 V4.2.3 发布说明

## 核心变化

- 后台主逻辑从 shell 高频轮询切换为原生守护进程 `oplus_smart_dimmingd`
- 前台应用检测优先使用 `cpuset/inotify` 事件驱动
- 保留低频 fallback 兜底，但只在长时间无事件时触发
- WebUI 新增运行诊断区，可直接查看：
  - 服务状态
  - 运行模式
  - 屏幕状态
  - 前台应用
  - 事件驱动 / 强制回退 / 监听数量 / 事件次数 / 回退轮询 / 解析失败

## 稳定性修复

- 修复安装后原生守护进程可执行权限丢失的问题
- 修复原生二进制在目标设备上的 C++ 运行时兼容问题
- 修复 KernelSU 模块页执行 / WebUI 双入口适配
- 修复执行按钮无明确输出的问题，现在会打印诊断信息、服务 PID 和重载结果
- 修复配置重载后未立即按当前前台应用重新计算状态的问题
- 恢复服务重复启动保护，避免 service launcher 被重复拉起造成多实例风险
- WebUI 新增 `重启服务` 按钮，便于排障时直接重启后台守护进程
- WebUI 状态刷新间隔调整为 5 秒
- 屏幕状态检测间隔调整为 3 秒
- 发布包仅保留运行期脚本，不再包含构建脚本
- 优化包名解析为空时的处理：
  - 先短延迟重试
  - 保留上一个有效包名，避免瞬时抖动
- 收紧 fallback 轮询策略：
  - 强制 fallback：15s
  - 常规无事件兜底：60s
  - `cpuset` 稳定后进一步降低到 180s

## 真机验证

验证设备：

- OnePlus PLK110 / OnePlus 15
- ColorOS 16.0.3.503

已验证通过：

- 普通应用保持默认状态 `2`
- 名单应用 `com.tencent.tmgp.sgame` 自动切换到经典状态 `0`
- 返回桌面后恢复默认状态 `2`
- 熄屏时进入 `screen_off` 流程并恢复默认状态
- 亮屏后重新识别前台应用并恢复 `event` 模式
- 重启后自动拉起原生守护进程
